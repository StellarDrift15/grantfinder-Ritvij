PRODUCT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GrantFinder NP is an AI-powered grant and sponsorship matching platform
designed to help non-profits and FIRST Robotics teams discover high-impact
funding tailored to their specific mission and requirements.

Core Value Proposition:
  - Eliminate hours of manual grant research with AI-curated matches
  - Confidence-ranked funding recommendations (0–100% match score)
  - Specialized tools for FIRST Robotics teams (FRC, FTC, FLL)
  - Corporate sponsorship finder with cold outreach guidance
  - Offline fundraising guide for local business cold calling

PHASE 1 — CORE GRANT MATCHING ENGINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Goal: Build the foundational grant discovery and AI matching system.

Entities Created:
  - Nonprofits: stores org profiles (name, EIN, focus area, budget, location, mission keywords)
  - FundingOpportunities: grant database (title, provider, type, value, deadline, sectors, robotics flag, URL)
  - MatchingResults: stores AI match output (search_id, funding_id, confidence score, reason)
  - SearchHistory: tracks when organizations ran scans
  - Grants: supplemental grant records

Pages Built:
  - Dashboard (main page): split-screen layout with org profile form (left) + results panel (right)

Key Features Implemented:
  - OrgProfileForm: captures nonprofit name, EIN, focus area, budget, location, mission keywords
  - AI matching via InvokeLLM (GPT-4o-mini → upgraded to gpt_5_mini)
  - 0–100 confidence score with visual progress bar per match
  - Results hidden until scan button clicked
  - Grant results sorted by confidence score descending
  - GrantMatchCard component with type badges, value display, match reasoning
  - Application URL links shown only when available
  - Loading spinner during AI scan

Design Decisions:
  - Inter font family throughout
  - Indigo color scheme for main grant finder
  - Split-screen layout (sidebar form + main results panel)
  - shadcn/ui + Tailwind CSS + framer-motion animations

Goal: Build a dedicated corporate sponsorship matching tool with AI.

New Entity: Sponsorships
  Fields: company_name, contact_email, contact_phone, contact_person,
          typical_amount, industry, description, website_url, accepts_robotics,
          sponsorship_status, target_programs, geographic_focus, community_notes

Status Types (ranked by priority):
  - Confirmed Sponsor, Open Grant, Email Outreach, Call Required,
    EDU Discounts, Needs Connection, Untried

New Page: /sponsorship-finder (SponsorshipFinder.jsx)
  - Purple color scheme (distinct from main grant finder indigo)
  - Same split-screen layout as Dashboard

New Components:
  - SponsorshipProfileForm: captures team name, number, program type, location,
    team size, years active, description, mentor/parent connections
  - SponsorshipResults: results panel with scanning/empty/no-results states
  - SponsorshipCard: displays company details, match score, contact info,
    action buttons (email, call, website)

AI Matching Logic:
  - Sends all sponsorship records + team profile to GPT (gpt_5_mini)
  - Status-based scoring priority:
      Open Grant / Confirmed Sponsor → 85–100
      Email Outreach → 70–90
      Call Required → 65–85
      EDU Discounts → 60–80
      Needs Connection → 50–70
      Untried → 55–75
  - Returns all sponsors scoring above 45

Outreach Ordering Rule:
  - When phone number is available → ALWAYS recommend calling first
  - Format: "Call [company] first at their main number, then follow up with email"
  - When no phone → recommend email outreach

PHASE 6 — MENTOR CONNECTION INTELLIGENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Goal: Help teams leverage internal company connections to unlock sponsorships.

Problem Solved:
  Teams often have mentors or parents who work at large companies
  (Boeing, Microsoft, Google, etc.) but don't know how to use that
  connection to unlock corporate matching programs or sponsorships.

Implementation — Frontend Pre-Matching (3 strategies for robustness):
  Strategy 1: Exact full company name match in mentor text
  Strategy 2: All significant words present (ignores stop words: Inc, LLC, Corp, etc.)
  Strategy 3: Any significant word as a whole word (regex boundary match)
  → mentorMatchedIds set populated before LLM call

LLM Instructions for Mentor Matches:
  - Score mentor-connected companies 90–100 regardless of status
  - has_mentor_connection = true
  - match_reason must include step-by-step internal guidance:
      1. How to ask the mentor to inquire at their company
      2. Who to contact internally (Community Relations, HR, CSR team)
      3. What to say and what to ask for
      4. Reference any volunteer grant/matching programs

Post-LLM Fallback Guarantee:
  - If LLM misses a mentor-matched company or scores it too low:
      → Injected manually at top with confidence 92
      → Pre-written internal approach guidance included
  - If LLM included it: boost confidence to max(existing, 90) and set flag

Sorting Logic:
  - Mentor connections always appear first
  - Then sorted by confidence score descending

Company Name Validation:
  - Added company_name field to LLM response schema
  - Post-processing: if match_reason doesn't mention the correct company name,
    prepend correction sentence automatically
